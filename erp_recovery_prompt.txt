## CRITICAL LESSONS LEARNED
1. Django-tenants migrations are fragile — when they fail, create tables MANUALLY via psql
2. The "django_content_type already exists" error blocks tenant migrations — mark as applied manually
3. save(update_fields=[...]) fails silently — generate QR inside serializer create() instead
4. HR page uses direct api.get() calls, NOT imports from services/api.js (avoids function name mismatches)
5. Each API call wrapped in safe() so one failure can't crash entire page
6. Frontend JS caching is aggressive — increment Dockerfile CACHEBUST arg and test in incognito
7. ${PORT:-8080} doesn't work in Railway — use sh -c "... $PORT ..."
8. WhiteNoise must be before tenant middleware in MIDDLEWARE order
9. Decimal('1/3') = 0 in Python — use Decimal('1') / Decimal('3')
10. POS invoices generate QR in _create_zatca_invoice() method, not in serializer
